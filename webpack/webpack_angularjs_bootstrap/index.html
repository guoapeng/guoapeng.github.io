说明

适用范围

  • 本教程适用于angularjs1.x 和bootstrap4.x

为什么要使用webpack

  • 传统web开发-编写html文件并手动引入js包, 已经不能适应现代web开发了, 具体体现在:
    1. 不能适应模块化编程的思想
    2. 不能很好的利用nodejs包
    3. 如果需要使用less, sass, typescript, es5, es6需要手动编译成源文件为css, js代码, 并打包.
    4. 为什么选择webpack, 相对于grunt 或 gulp 加bower组合, webpack能更好的处理现代web开发中的痛点. 另外bower的官网已经推荐使用yarn+webpack组合了.
  • 当然好处还有很多, 这里只是简单列举这些.

配置webpack dev server

为什么要安装dev server?
如果没有dev server, 我们每次修改后都要运行webpack去打一次包, 才能看到改动后的效果. 如果开启dev server, 修改可以及时反映到界面上, 不用我们手动打包, 再点击运行. 方便开发.
